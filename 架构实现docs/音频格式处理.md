# 音频格式处理设计

## 1. 支持的音频格式

根据需求，系统默认支持并仅支持以下音频格式：

- **格式**：WAV和PCMA（A-law）
- **采样率**：16kHz和8kHz
- **通道数**：单声道（mono）

## 2. 音频格式对处理流程的影响

不同的音频格式会影响音频处理器和缓冲区的逻辑，主要体现在以下几个方面：

### 2.1 数据表示差异

- **WAV (PCM)**：通常是未压缩的线性PCM数据，可以是16位整数（int16）或32位浮点数（float32）
- **PCMA (A-law)**：是一种压缩格式，每个样本使用8位表示，需要解压缩为线性PCM才能进行处理

### 2.2 采样率影响

- **缓冲区大小计算**：不同采样率下，相同时长的音频对应不同数量的样本
- **处理块大小**：16ms的音频块在16kHz下是256个样本，在8kHz下是128个样本
- **VAD性能**：VAD算法可能对特定采样率有优化，可能需要重新训练或调整参数

### 2.3 数据类型考虑

- **内存使用**：不同数据类型占用不同大小的内存（int16: 2字节/样本，float32: 4字节/样本）
- **计算效率**：不同数据类型的计算效率不同，浮点运算通常更慢但精度更高
- **动态范围**：int16的动态范围是±32767，float32可以表示更大的动态范围

## 3. 音频格式处理方案

### 3.1 优化的格式转换层

在音频处理器和缓冲区之前添加一个格式转换层，负责将不同格式的音频统一转换为内部处理格式：

```mermaid
graph LR
    A[输入音频] --> B[格式检测]
    B --> C{格式类型}
    C -->|WAV| D[WAV处理器]
    C -->|PCMA| E[PCMA处理器]
    D --> F[统一格式转换]
    E --> F
    F -->|float32| G[音频缓冲区]
    G --> H[音频处理器]
    H --> I[VAD引擎]
```

- **统一内部格式**：选择float32作为内部处理格式，提供足够的精度和动态范围
- **专用格式处理器**：为每种支持的格式提供专用处理器，优化转换性能
- **延迟转换**：在处理流程的最早阶段进行格式转换，避免重复转换
- **零拷贝原则**：尽可能使用视图而非复制进行格式转换

```python
from enum import Enum
from pydantic import BaseModel, Field
import numpy as np
from typing import Tuple, Union, Optional

class AudioFormat(str, Enum):
    """支持的音频格式"""
    WAV = "wav"
    PCMA = "pcma"

class SampleRate(int, Enum):
    """支持的采样率"""
    RATE_8K = 8000
    RATE_16K = 16000

class AudioFormatConfig(BaseModel):
    """音频格式配置"""
    format_type: AudioFormat = Field(description="音频格式类型")
    sample_rate: SampleRate = Field(description="采样率")
    channels: int = Field(default=1, description="通道数")
    dtype: str = Field(default="float32", description="数据类型")

class FormatConverter:
    """音频格式转换器"""
    def __init__(self):
        # 注册格式处理器
        self.format_handlers = {
            AudioFormat.WAV: self._handle_wav,
            AudioFormat.PCMA: self._handle_pcma
        }
        
    def convert_to_internal_format(self, audio_data, config: AudioFormatConfig) -> Tuple[np.ndarray, int, int]:
        """
        将输入音频转换为内部处理格式
        
        Args:
            audio_data: 输入音频数据
            config: 音频格式配置
            
        Returns:
            转换后的音频数据，采样率，通道数
        """
        # 检查格式是否支持
        if config.format_type not in self.format_handlers:
            raise ValueError(f"Unsupported audio format: {config.format_type}")
            
        # 调用对应的格式处理器
        handler = self.format_handlers[config.format_type]
        return handler(audio_data, config)
        
    def _handle_wav(self, audio_data, config: AudioFormatConfig) -> Tuple[np.ndarray, int, int]:
        """处理WAV格式"""
        # 确保数据类型为float32
        if audio_data.dtype != np.float32:
            # 如果是int16，归一化到[-1.0, 1.0]范围
            if audio_data.dtype == np.int16:
                audio_data = audio_data.astype(np.float32) / 32767.0
            else:
                audio_data = audio_data.astype(np.float32)
                
        return audio_data, config.sample_rate, config.channels
        
    def _handle_pcma(self, audio_data, config: AudioFormatConfig) -> Tuple[np.ndarray, int, int]:
        """处理PCMA格式"""
        # 将PCMA解压缩为线性PCM
        pcm_data = self._pcma_to_pcm(audio_data)
        
        # 转换为float32
        float_data = pcm_data.astype(np.float32) / 32767.0
        
        return float_data, config.sample_rate, config.channels
        
    def _pcma_to_pcm(self, pcma_data):
        """PCMA到PCM的转换"""
        # 使用优化的转换算法
        # 这里可以使用查找表或向量化操作提高性能
        return self._pcma_to_pcm_optimized(pcma_data)
        
    def _pcma_to_pcm_optimized(self, pcma_data):
        """优化的PCMA到PCM转换"""
        # 使用预计算的查找表加速转换
        # 实际实现会使用NumPy的向量化操作
        return np.zeros_like(pcma_data, dtype=np.int16)  # 占位实现
```

## 5. 音频处理器适配

### 5.1 格式感知处理

修改音频处理器，使其能够处理不同格式的音频：

```python
class AudioProcessor:
    def __init__(self, vad_engine, supported_formats=None):
        """
        初始化音频处理器
        
        Args:
            vad_engine: VAD引擎
            supported_formats: 支持的格式配置
        """
        self.vad_engine = vad_engine
        
        # 默认支持的格式
        self.supported_formats = supported_formats or {
            "formats": ["wav", "pcma"],
            "sample_rates": [8000, 16000],
            "channels": [1]
        }
        
    async def process_chunk(self, audio_data, format_type, sample_rate, channels):
        """
        处理音频块
        
        Args:
            audio_data: 音频数据
            format_type: 音频格式类型
            sample_rate: 采样率
            channels: 通道数
            
        Returns:
            处理结果
        """
        # 检查格式是否支持
        if format_type not in self.supported_formats["formats"]:
            raise ValueError(f"Unsupported audio format: {format_type}")
            
        if sample_rate not in self.supported_formats["sample_rates"]:
            raise ValueError(f"Unsupported sample rate: {sample_rate}")
            
        if channels not in self.supported_formats["channels"]:
            raise ValueError(f"Unsupported channels: {channels}")
        
        # 转换为内部处理格式
        internal_data, internal_rate, internal_channels = convert_to_internal_format(
            audio_data, format_type, sample_rate, channels
        )
        
        # 计算块大小和重叠大小
        frame_duration_ms = 16  # 16ms per frame for VAD
        frame_size = calculate_chunk_size(frame_duration_ms, internal_rate)
        
        # 分割为VAD帧
        num_frames = len(internal_data) // frame_size
        
        # 处理每一帧
        results = []
        for i in range(num_frames):
            frame_data = internal_data[i * frame_size: (i + 1) * frame_size]
            result = await self.vad_engine.process_frame(frame_data)
            results.append(result)
            
        return results
```

#### 7.1.1 预计算块大小

```python
from functools import lru_cache
from enum import Enum
from typing import Dict, Tuple

class ChunkSizeCache:
    """块大小缓存"""
    def __init__(self):
        # 预计算常用块大小
        self._cache = {
            SampleRate.RATE_8K: {  # 8kHz
                16: 128,    # 16ms
                32: 256,    # 32ms
                250: 2000,  # 250ms
            },
            SampleRate.RATE_16K: {  # 16kHz
                16: 256,    # 16ms
                32: 512,    # 32ms
                250: 4000,  # 250ms
            }
        }
    
    @lru_cache(maxsize=32)
    def get_chunk_size(self, duration_ms: int, sample_rate: SampleRate) -> int:
        """获取块大小，使用缓存加速"""
        # 先查找预计算值
        if sample_rate in self._cache and duration_ms in self._cache[sample_rate]:
            return self._cache[sample_rate][duration_ms]
        
        # 计算并缓存
        size = int(duration_ms * sample_rate / 1000)
        if sample_rate not in self._cache:
            self._cache[sample_rate] = {}
        self._cache[sample_rate][duration_ms] = size
        return size
```

```python
class MemoryOptimizer:
    """内存优化器"""
    def __init__(self):
        # 检测CPU支持的SIMD指令集
        self.simd_support = self._detect_simd_support()
        
    def _detect_simd_support(self) -> Dict[str, bool]:
        """检测CPU支持的SIMD指令集"""
        # 实际实现会检测CPU特性
        return {
            "avx": True,
            "avx2": False,
            "sse": True,
            "sse2": True
        }
        
    def optimize_buffer(self, shape: Tuple[int, ...], dtype: np.dtype) -> np.ndarray:
        """创建优化的缓冲区"""
        alignment = self._get_optimal_alignment(dtype)
        return np.zeros(shape, dtype=dtype, order='C', align=alignment)
        
    def _get_optimal_alignment(self, dtype: np.dtype) -> int:
        """获取最优的内存对齐大小"""
        if dtype == np.float32 and self.simd_support.get("avx", False):
            return 32  # AVX指令集需要32字节对齐
        elif dtype == np.int16 and self.simd_support.get("sse", False):
            return 16  # SSE指令集需要16字节对齐
        else:
            return 8   # 默认对齐
```

## 8. 格式处理器注册机制

为了支持更灵活的格式扩展，我们可以实现一个格式处理器注册机制：

```python
from typing import Dict, Callable, Type, Any

class FormatProcessorRegistry:
    """格式处理器注册中心"""
    def __init__(self):
        self._processors: Dict[AudioFormat, Type[BaseFormatProcessor]] = {}
        
    def register(self, format_type: AudioFormat, processor_class: Type['BaseFormatProcessor']):
        """注册格式处理器"""
        self._processors[format_type] = processor_class
        
    def get_processor(self, format_type: AudioFormat) -> 'BaseFormatProcessor':
        """获取格式处理器实例"""
        if format_type not in self._processors:
            raise ValueError(f"No processor registered for format: {format_type}")
            
        return self._processors[format_type]()
        
    def supported_formats(self) -> list:
        """获取支持的格式列表"""
        return list(self._processors.keys())

class BaseFormatProcessor:
    """格式处理器基类"""
    def convert_to_internal(self, audio_data: np.ndarray, config: AudioFormatConfig) -> np.ndarray:
        """转换为内部格式"""
        raise NotImplementedError("Subclasses must implement this method")
        
    def convert_from_internal(self, audio_data: np.ndarray, config: AudioFormatConfig) -> np.ndarray:
        """从内部格式转换"""
        raise NotImplementedError("Subclasses must implement this method")

# 使用示例
registry = FormatProcessorRegistry()

class WAVProcessor(BaseFormatProcessor):
    """WAV格式处理器"""
    def convert_to_internal(self, audio_data, config):
        # WAV转换逻辑
        return audio_data.astype(np.float32) / 32767.0
        
    def convert_from_internal(self, audio_data, config):
        # 内部格式转WAV
        return (audio_data * 32767.0).astype(np.int16)

class PCMAProcessor(BaseFormatProcessor):
    """PCMA格式处理器"""
    def __init__(self):
        self.converter = PCMAConverter()
        
    def convert_to_internal(self, audio_data, config):
        # PCMA转换逻辑
        pcm_data = self.converter.pcma_to_pcm(audio_data)
        return pcm_data.astype(np.float32) / 32767.0
        
    def convert_from_internal(self, audio_data, config):
        # 内部格式转PCMA
        pcm_data = (audio_data * 32767.0).astype(np.int16)
        return self.converter.pcm_to_pcma(pcm_data)

# 注册处理器
registry.register(AudioFormat.WAV, WAVProcessor)
registry.register(AudioFormat.PCMA, PCMAProcessor)
```

## 9. 总结

不同的音频格式确实会影响音频处理器和缓冲区的逻辑，但通过合理的设计，可以使系统灵活地处理不同格式的音频。关键点包括：

1. **格式转换层**：在处理前将不同格式统一转换为内部处理格式
2. **专用格式处理器**：为每种支持的格式提供专用处理器，优化转换性能
3. **格式处理器注册机制**：支持灵活扩展新格式
4. **缓存与预计算**：使用缓存和预计算提高处理效率
5. **内存优化**：根据不同格式优化内存布局和访问模式

通过这些设计，系统可以高效地支持WAV和PCMA格式，16kHz和8kHz采样率，单声道的音频数据，同时保持低延迟和高性能。系统也可以方便地扩展支持其他音频格式。
